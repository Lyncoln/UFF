Apache Spark

Antes de falarmos sobre sparklyr vamos apresentar o Spark. O Apache Spark um mecanismo de anlise unificada para processamento de dados em larga escala.

Sparklyr

O sparklyr um pacote que fornece uma interface entre o R e o Apache Spark. O sparklyr surgiu a pedido da comunidade do R querendo uma interface dplyr nativa para o Spark. O sparklyr tambm fornece interfaces para os algoritmos de aprendizado de mquina distribuda do Spark entre outros. Como destaques temos:

  • Manipule de forma interativa os dados do Spark usando dplyr e SQL (via DBI).

  • Filtrar e agregar conjuntos de dados do Spark e, em seguida, traz-los para o R para anlise e visualizao.

  • Organize o aprendizado de mquina distribudo a partir do R usando o Spark MLlib ou o H2O SparkingWater .

  • Crie extenses que chamam a API Spark completa e forneam interfaces para os pacotes do Spark.

  • Suporte integrado para estabelecer conexes Spark e navegar por quadros de dados do Spark no IDE do RStudio.

Instalao

Para instalao do pacote vamos usar a funo install.packages()

install.packages("sparklyr")
library(sparklyr) # Carregando o pacote

Para utilizar o sparklyr e suas funes precisamos ter instalado tambm o spark. Para instalar o spark usaremos a funo spark_install() do prprio pacote sparkly.

spark_install(version = "2.1.0")

Observao: Tambm preciso ter instalado o Java no computador em sua verso mais recente. O Java pode ser baixado no seguinte link https://www.java.com

Conectando-se ao Spark

Voc pode se conectar a ambas as instncias locais do Spark, bem como aos clusters remotos do Spark.Para isso necessrio usar o Rstudio Server ou o Pro.

Existem Clusters experimentais, como o Livy (https://livy.incubator.apache.org/).Aqui vamos nos conectar a uma instncia local:

sc <- spark_connect(master = "local")

A conexo Spark retornada sc fornece uma fonte de dados dplyr remota conectada ao Spark.

dplyr um pacote R para trabalhar com dados estruturados dentro e fora de R. dplyr torna a manipulao de dados para usurios R fcil, consistente e de alto desempenho. Com o dplyr como uma interface para manipular o Spark DataFrames, voc pode selecionar, filtrar e agregar dados, usar as funes da janela (por exemplo, para amostragem), executar junes em DataFrames e coletar dados do Spark em R.

Lendo dados

Voc pode ler dados no Spark DataFrames usando as seguintes funes:

  • spark_read_csv: l um arquivo CSV e fornece uma fonte de dados compatvel com dplyr
  • spark_read_json: l um arquivo JSON e fornece uma fonte de dados compatvel com dplyr
  • spark_read_parquet: l um arquivo em parquet e fornece uma fonte de dados compatvel com dplyr

Independentemente do formato dos dados, o Spark oferece suporte leitura de dados de diversas fontes de dados diferentes. Isso inclui dados armazenados em HDFS ( hdfs://protocolo), Amazon S3 ( s3n://protocolo) ou arquivos locais disponveis para os ns de trabalho do Spark ( file://protocolo)

Cada uma dessas funes retorna uma referncia a um Spark DataFrame que pode ser usado como uma tabela dplyr (tbl).

Dados

Para o exemplo usaremos como base os dados do nycflights13, pacote do R. Este pacote contm dados para todos os 336.776 voos que partiram de Nova York em 2013. Ele tambm inclui metadados teis sobre companhias areas, aeroportos, clima e avies. Os dados vm do Bureau de Estatsticas de Transporte dos EUA e esto documentados em ?nycflights13.

install.packages("nycflights13") # Instalando o pacote nycflights13
library(nycflights13) # Carregando o pacote

Aps conectados ao cluster (no caso local), copie os dados dos voos usando a funo copy_to.

Advertncia: Os dados de voo nycflights13 so convenientes para demonstraes dplyr porque so pequenos, mas na prtica, dados grandes raramente devem ser copiados diretamente de objetos R.

library(dplyr) # Carregando o pacote
flights <- copy_to(sc, flights, "flights")

A varivel flights representa no do R, o data frame dentro da conexo sc com o Spark, com ela podemos vizualizar uma parte dos dados(mximo de 1000 linhas) e manipul-los.

Verbos DPLYR

Os verbos so comandos dplyr para manipular dados. Quando conectado a um Spark DataFrame, dplyr converte os comandos em instrues SQL do Spark. Fontes de dados remotas usam exatamente os mesmos cinco verbos que as fontes de dados locais. Aqui esto os cinco verbos com seus comandos SQL correspondentes:

  • select ~ SELECT
  • filter ~ WHERE
  • arrange ~ ORDER
  • summarise ~ aggregators: sum, min, sd, etc.
  • mutate ~ operators: +, *, log, etc.

Por exemplo, veja o seguinte cdigo:

c1 <- filter(flights, day == 17, month == 5) %>% 
  select(c1, year, month, day, air_time) %>% 
  arrange(c2, year, month, day) %>% 
  mutate(c3, air_time_hours = air_time / 60)

Coletando para o R

carrierhours <- collect(c1)

ou diretamente

c1 <- filter(flights, day == 17, month == 5) %>%
  select(c1, year, month, day, air_time) %>%
  arrange(c2, year, month, day) %>% 
  mutate(c3, air_time_hours = air_time / 60) %>%
  collect()

collect() executa a consulta Spark e retorna os resultados para R para anlise e visualizao adicionais.

Exemplo em sala de aula:

#Retornar um objeto do tipo "spark"

df1 <- flights  %>% 
  filter(month == 11 , day == 13)

#Retornar um objeto do tipo "tibble"
  
df2 <- flights  %>% 
  filter(month == 11 , day == 13) %>% 
  collect()

Desconectando:

spark_disconnect(sc)

Concluindo

Como visto nos exemplos acima trabalhamos localmente, porm o objetivo do pacote sparklyr trabalhar com um fluxo de dados (big data), ou seja, trabalhar em rede (cluster). Fazendo corretamente a conexo ao local do cluster atravs da funo spark_connect() trabalharemos basicamente com as funes do pacote dplyr.

Como podemos ver na imagem, o sparklyr faz o papel de ponte para acessar os dados no data lake (dados armazenados) e permitir trabalhar com os pacotes no R.
